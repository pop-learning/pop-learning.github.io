Coding for HTML Email

Avoid shorthand CSS

Avoid shorthand CSS, e.g. typing #CCC instead of #CCCCCC to define a hex colour. Shorthand CSS can result in unpredictable HTML email rendering. Be as specific as possible in your CSS. See the example shown to your right.

Avoid block level elements

Block level elements such as <p> and <h1> tags can be either stripped out of your code or inherit unexpected and unwanted styles. Instead, enclose your content in <td> cells, and use inline CSS to style the content.

Avoid CSS positioning, floats, and clears

Email clients do not support these CSS properties, and they should be avoided. Instead use tables to position your content.
