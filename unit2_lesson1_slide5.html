Problem: Inherited padding on adjoining <td> cells


The top-right image represents two HTML table cells. The left <td> cell has 25 pixels of padding applied top, left, and right of the table cell. The right table cell has no pixels of padding applied. This is how the padding appears when the email is viewed in the browser window.

The middle-right image represents how the same two table cells appear when the email is viewed in Outlook. Padding that was applied to the top of the left <td> cell has also been applied to the top of the right <td> cell. This is an example of how unwanted padding can appear in Outlook.

Solution:Nest a table inside the left <td> cell and apply the padding to this inner table.

The bottom-right image represents a nested table placed inside the left <td> cell. As the nested table is separate to the outer container table, no unwanted padding will be applied to right <td> cell of the outer table.
